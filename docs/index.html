VaultForge v3.0.0

从生成工具到知识系统

v1.2.0 是一个生成工具——输入 PDF，输出笔记。它的核心假设是"一次性加工"：一次运行处理一批材料，产出即终态。工具本身不记忆任何状态，不区分生成内容与用户编辑，不做生成后的独立质量校验。

v3.0.0 是系统级的重新设计。它在三个维度上同时改变了工具的行为方式——不是增量迭代，而是将学习工具从"文本生成器"重新定义为认知过程的工程支撑系统。

Pillar 1

运行时可靠性

中断恢复和写入保护使处理流程不再脆弱。状态机 + 原子写确保任何时刻系统状态可恢复。

Pillar 2

知识积累模型

从一次性加工变为可检测、可增量、可保护的持续建构。Phase 0 感知已有知识，三态 ACL 保护用户编辑。

Pillar 3

输出精度控制

从关键词匹配升级为五类逻辑关系分类和独立质量审查回路。三阶段漏斗在精度和成本间取得平衡。

Pillar 1

运行时可靠性

v1.2.0

无状态运行时

Agent 崩溃或网络中断 → 所有进度丢失。已写入文件可能不完整。重新触发后从第一页重读。

v3.0.0

五态状态机 + 原子写入

每个笔记携带 status 字段。filling 状态拆分为两种崩溃子场景，分别恢复。.md.tmp → verify → rename 确保文件完整性。

与学习理论的关系
分布式学习（Spaced Learning）的记忆保持率显著高于集中学习，但前提是中断后能无缝恢复。不支持断点恢复的工具实际上在惩罚有效的学习策略。状态机将学习过程从连续作业解耦为可中断的会话——这是将间隔效应转化为工程约束。
Pillar 2

知识积累模型

v1.2.0

线性处理

读取所有文件 → 生成路线图 → 创建所有笔记 → 建立所有链接。新文件加入时，整流程重跑，已有笔记被覆盖或丢弃。

v3.0.0

Phase 0 · 三态保护

Phase 0 在内容生成前扫描已有笔记状态。Pristine 可刷新 · User-modified 受保护 · Locked 完全冻结。增量模式下仅处理新文件，已有文件原样保留。

状态判定方式系统行为
Pristine生成后未被修改可自动刷新内容
User-modified检测到 mtime 与 date 差异受保护，自动操作不覆盖
Locked用户手动标记系统完全忽略

增量模式下的链接策略

新笔记 ↔ 新笔记：链接直接写入。新笔记 → 已有笔记：仅输出建议 JSON，不修改已有文件。已有笔记的链接网络是使用者已建立的心理模型——新增链接应作为可选数据源，而非自动变更。

与学习理论的关系
Bruner 的螺旋课程模型证明，最有效的知识建构是在不同时间以更高复杂度重新审视同一主题——但前提是每次返回时已有结构不被重建。Phase 0 + 增量处理 + 三态保护，使 VaultForge 从一次性工具变为螺旋式学习系统。
Pillar 3

输出精度控制

v1.2.0

关键词匹配

两个笔记提到相同词汇即建立关联。停留在识别记忆层——"我见过这个词"，而非"这些概念如何关联"。

v3.0.0

五类逻辑关系 + 六维审查

推导 · 类比 · 矛盾 · 应用 · 背景 —— 五类关系覆盖 Bloom 分类学从理解到评价的全部认知层次。note‑reviewer 在六个维度上独立审查每篇笔记。

与学习理论的关系
Kluger & DeNisi (1996) 的元分析确认，有效反馈需三个条件：针对任务而非自我评价、提供具体修正方向、给予修正机会。六维审查 + 自动重写回路构成工程闭环——反馈精确到文件路径和行号，方向按 HIGH / MEDIUM 分级，修正在同一 session 内完成。

开始使用

git clone https://github.com/Easonnotsing/VaultForge.git ~/.agents/skills/VaultForge
pip install pypdf

在 Claude Code、Codex 或 Cursor 中触发后，首先选择输出语言（English / 中文）。系统自动运行 Phase 0 扫描，如已有知识库，将询问增量更新还是全新生成。
